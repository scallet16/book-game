# PRD: 수다쟁이 여니쌤의 뿜뿜 놀이터

## 1. 제품 목적
`수다쟁이 여니쌤의 뿜뿜 놀이터`는 특수교육의 세심한 설계를 바탕으로, 다양한 학습자가 놀면서 읽고, 말하고, 쓰고, 생각하고, 표현하는 경험을 제공하는 보편적 학습 경험 플랫폼이다.

이 프로젝트는 게임을 만드는 것이 목적이 아니다. 게임형 조작은 학생의 배움을 돕는 도구로만 사용한다.

이 플랫폼은 특정 수준의 학생만을 대상으로 하지 않는다. 중증장애학생, 경증장애학생, 느린학습자, 유아, 초등 저학년 학생, 통합학급 학생, 교사가 같은 책과 같은 주제 안에서 각자 가능한 방식으로 참여할 수 있어야 한다.

## 2. 핵심 사용자
- 특수교육 학생
- 중증장애학생
- 경증장애학생
- 느린학습자
- 유아
- 초등 저학년 학생
- 통합학급 학생
- 전자칠판 앞에서 함께 활동하는 교사와 학생
- 휴대폰 또는 태블릿으로 활동에 참여하는 학생
- 향후 활동을 등록하고 관리하는 교사

## 3. 제품 원칙
- 학생은 화면을 보고 3초 안에 무엇을 해야 하는지 알 수 있어야 한다.
- 긴 글을 읽게 하지 않는다.
- 큰 버튼, 큰 그림, 간단한 문장, 직관적인 조작을 우선한다.
- 예쁜 디자인보다 학생이 사용하기 쉬운 디자인을 우선한다.
- 전자칠판에서 30분 이상 사용해도 눈이 피곤하지 않아야 한다.
- 애니메이션은 학습을 방해하지 않는 범위에서만 사용한다.
- 학생이 "공부한다"보다 "놀러 간다"고 느끼게 한다.
- 놀다 보면 읽기, 말하기, 쓰기, 생각하기, 표현하기가 자연스럽게 일어나야 한다.
- 같은 책, 같은 주제 안에서 다양한 학습자가 각자 가능한 방식으로 참여할 수 있어야 한다.
- 화면에는 `초급`, `고급`, `중증`, `경증`처럼 낙인으로 보일 수 있는 표현을 사용하지 않는다.
- 활동은 쉬운 것에서 어려운 것 순서로 자연스럽게 배치하되, 학생이 스스로 활동을 선택할 수 있게 한다.

## 4. 보편적 설계 원칙
- 글을 읽기 어려운 학생도 사용할 수 있어야 한다.
- 큰 아이콘, 음성 안내, 짧은 문장, 상징 단서를 제공한다.
- 조작이 어려운 학생도 사용할 수 있어야 한다.
- 큰 버튼, 넓은 터치 영역, 전자칠판과 휴대폰 모두에서 누르기 쉬운 배치를 사용한다.
- 드래그가 어려운 학생을 위해 가능하면 클릭 또는 선택 대안을 제공한다.
- 읽고 쓰기가 가능한 학생도 도전할 수 있어야 한다.
- 문장 만들기, 키보드 입력, 내 생각 말하기, 이야기 확장 활동을 제공한다.
- 유아와 초등 저학년 학생도 흥미를 느낄 수 있어야 한다.
- 게임성, 선택 경험, 캐릭터, 칭찬 피드백은 학습을 돕는 범위에서 사용한다.
- 교사가 수업에 활용하기 쉬워야 한다.
- 책별 활동 구조가 명확하고, 활동이 읽기/쓰기/게임/말하기 영역으로 분류되며, 교육 목적이 분명해야 한다.

## 5. 시각 디자인 원칙
- 파스텔톤
- 낮은 채도
- 큰 여백
- 둥근 버튼
- 큰 글씨
- 부드러운 배경
- 구름, 별, 말랑한 젤리 느낌
- 귀여운 말랑이 캐릭터 분위기
- 시선을 빼앗는 장식보다 행동을 돕는 화면 구성을 우선한다.

## 6. 말랑이 원칙
- 말랑이는 정답을 알려주는 캐릭터가 아니다.
- 말랑이는 학생이 스스로 찾을 수 있도록 힌트를 주는 학습 친구다.
- 말랑이는 속도를 칭찬하지 않는다.
- 말랑이는 정답만 칭찬하지 않는다.
- 말랑이는 도전, 집중, 끝까지 하기, 다시 시도하기, 표현하려는 시도를 칭찬한다.

## 7. 활동 설계 원칙
- 같은 활동을 반복 생산하지 않는다.
- 책마다 핵심 개념을 분석한 뒤 가장 적합한 활동을 선택한다.
- 같은 조작 방식이라도 책마다 내용, 질문, 그림, 표현 활동은 달라져야 한다.
- 활동은 재미를 위한 장치가 아니라 배움을 돕는 장치여야 한다.
- 활동 화면은 가능한 한 한 가지 행동만 요구한다.
- 책마다 `book.json`의 `bookType`과 `activitySections`를 기준으로 활동 영역 이름과 개수를 달리할 수 있다.
- 글이 적거나 글 없는 그림책에서도 읽기와 쓰기는 사라지지 않고, 장면 상상하기, 이야기 만들기, 문장 만들기, 말로 표현하기, 내 이야기 녹음하기로 형태가 바뀐다.
- 활동 예시는 `크게 보기`, `듣고 고르기`, `따라 말하기`, `따라 쓰기`, `직접 쓰기`, `문장 만들기`, `내 생각 말하기`처럼 행동 중심으로 표현한다.

## 8. 표현 원칙
사용하지 않을 표현:

- 초급
- 고급
- 중증용
- 경증용
- 쉬운반
- 어려운반
- 못했어요
- 틀렸어요

사용할 표현:

- 먼저 해 볼 활동
- 더 생각해 볼 활동
- 말로 해 보기
- 손으로 해 보기
- 듣고 골라 보기
- 내 생각 말하기
- 다시 도전하기
- 끝까지 해 보기

## 9. 저작권 보호 원칙
- 그림책 본문 이미지, 원화, 스캔 이미지는 사용하지 않는다.
- 학생 화면에는 책 제목, 작가, 출판사 정보만 표시한다.
- 활동에 필요한 이미지는 자체 제작 아이콘, 말랑이 캐릭터, 추상 장면, 상징 그림, 교사가 직접 만든 이미지로 대체한다.
- `그림 읽기`, `그림을 보고`처럼 본문 그림을 직접 보여주는 의미로 오해될 수 있는 표현은 학생 화면에서 사용하지 않는다.
- 필요한 경우 `장면을 떠올리고`, `상황을 상상하고`, `어울리는 말을 만들어 보고` 같은 표현을 사용한다.

## 10. 책 유형과 활동 영역
책별 `book.json`은 다음 확장 구조를 가진다.

글이 있는 그림책 예:

```json
{
  "id": "ppajida",
  "title": "빠질 때가 됐어",
  "bookType": "text-picture",
  "activitySections": [
    {
      "id": "game",
      "label": "놀이 활동",
      "enabled": true
    },
    {
      "id": "write",
      "label": "쓰기 활동",
      "enabled": true
    },
    {
      "id": "speak",
      "label": "읽고 말하기",
      "enabled": true
    },
    {
      "id": "dictionary",
      "label": "말랑 낱말사전",
      "enabled": true
    }
  ]
}
```

『빠질 때가 됐어』에서는 읽기와 녹음 영역을 `읽고 말하기`로 통합한다. `말랑 낱말사전`은 점수나 정답 판정 없이 교사와 학생이 함께 뜻을 살펴보는 설명형 영역으로 운영한다.

글 없는 그림책 예:

```json
{
  "id": "wordless-book-example",
  "title": "글 없는 그림책 예시",
  "bookType": "wordless-picture",
  "activitySections": [
    {
      "id": "scene-imagine",
      "label": "장면 상상하기",
      "enabled": true
    },
    {
      "id": "story-making",
      "label": "이야기 만들기",
      "enabled": true
    },
    {
      "id": "sentence-making",
      "label": "문장 만들기",
      "enabled": true
    },
    {
      "id": "voice",
      "label": "내 이야기 녹음하기",
      "enabled": true
    }
  ]
}
```

## 11. 1차 MVP 범위
1차 MVP는 플랫폼 완성이 아니라 웹사이트 동작 확인 버전이다.

필수 구현:
- 메인 화면
- 경제 놀이터 목록 페이지
- 책 놀이터 목록 페이지
- 한글 놀이터 목록 페이지
- 숫자 놀이터 목록 페이지
- 사회성 놀이터 진입 카드
- 생활 놀이터 진입 카드
- 책 놀이터 안에 `빠질 때가 됐어` 책 활동 연결
- 학생 화면에서 관리자 메뉴 숨김
- `/admin` 직접 접속용 관리자 화면 뼈대

1차 MVP에서 하지 않는 것:
- Firebase 실제 연결
- DB 저장
- AI 자동생성
- 실제 로그인 서버
- QR 코드 실제 생성
- 활동 자동 배포

## 13. 메인 화면 요구사항
- 상단에 작은 여니쌤 캐릭터와 브랜드 문구를 표시한다.
- 브랜드 문구: `수다쟁이 여니쌤의 뿜뿜 놀이터`
- 여니쌤 캐릭터는 로고처럼 작게만 사용한다.
- 캐릭터는 제목보다 튀지 않아야 한다.
- 메인은 설명하는 화면이 아니라 선택하는 화면이다.
- 학생의 시선은 큰 그림, 놀이터 이름, 선택 순서로 흐르게 한다.
- 메인에는 6개 큰 카드가 있어야 한다.
  - 책 놀이터
  - 경제 놀이터
  - 한글 놀이터
  - 숫자 놀이터
  - 사회성 놀이터
  - 생활 놀이터
- 홈 카드에는 긴 설명을 넣지 않는다.
- 각 버튼은 해당 목록 페이지로 이동한다.
- 학생 화면에는 관리자 버튼을 노출하지 않는다.

## 14. 관리자 화면 요구사항
관리자 화면은 `/admin` 경로로 직접 접근한다.

1차 MVP:
- 간단한 비밀번호 입력 화면
- 예시 비밀번호: `teacher1234`
- 비밀번호가 맞으면 관리자 화면 뼈대를 보여준다.
- 비밀번호가 틀리면 접근하지 못하게 한다.

향후 기능:
- 책 놀이터에 새 책 추가
- 책 제목, 작가, 출판사 입력
- 핵심 지도 내용 입력
- 넣고 싶은 활동 선택
- 공개/비공개 설정
- 생성된 활동 링크 확인
- QR 코드 생성 영역
- 기존 책 수정/삭제

## 15. 기술 방향
- 배포: Vercel 예정
- DB/인증/저장소: Firebase 예정
- 스타일링: Tailwind CSS 우선
- MVP는 정적 HTML/CSS/JS와 Tailwind CDN으로 빠르게 확인한다.
- 폰트: Pretendard

## 16. 보안 기준
- 학생 화면에 관리자 진입 버튼을 만들지 않는다.
- `/admin`은 직접 주소로 들어갈 때만 접근한다.
- MVP 비밀번호는 실제 보안 기능이 아니라 데모용임을 명확히 한다.
- Firebase API 키, 서비스 계정, 관리자 토큰 등 비밀값을 저장하지 않는다.
- 실제 DB 쓰기/삭제 기능은 Firebase Auth와 Security Rules 설계 후 구현한다.
- 공개/비공개 설정은 실제 접근 제어로 오해되지 않게 한다.

## 17. 현재 상태
- 1차 MVP 화면 구조는 생성되어 있다.
- 기존 `빠질 때가 됐어` HTML은 `books/ppajida/`에 연결되어 있다.
- 기존 활동 HTML은 MVP 연결용이며, 다음 단계에서 이 PRD의 학습 원칙에 맞게 재정리해야 한다.
