# 수다쟁이 여니쌤의 뿜뿜 놀이터 계획

> 개발 기준 문서는 `docs/PRD.md`다. `plan.md`는 PRD를 실행하기 위한 단계별 작업 계획으로 유지한다.

## 1차 MVP 목표
- 메인 화면에서 4개 놀이터로 이동한다.
- 경제, 책, 한글, 숫자 놀이터 목록 페이지가 열린다.
- 책 놀이터에서 `빠질 때가 됐어` 책 활동으로 이동한다.
- 학생 화면에는 관리자 버튼을 노출하지 않는다.
- 관리자, Firebase, DB, AI 자동생성은 다음 단계 TODO로 남긴다.

## 교육 방향
- 이 프로젝트는 게임 제작 프로젝트가 아니라 특수교육 학생이 놀면서 읽고, 말하고, 쓰고, 생각하고, 표현하는 경험을 제공하는 학습 공간이다.
- 재미는 목적이 아니라 배움을 돕는 도구다.
- 학생은 화면을 본 뒤 3초 안에 무엇을 해야 하는지 알 수 있어야 한다.
- 긴 설명보다 큰 버튼, 큰 그림, 간단한 문장, 직관적인 조작을 우선한다.
- 전자칠판에서 30분 이상 사용해도 눈이 피곤하지 않도록 낮은 채도, 파스텔톤, 큰 여백, 큰 글씨를 유지한다.
- 예쁜 디자인보다 아이들이 사용하기 쉬운 디자인을 우선한다.
- 애니메이션은 학습 집중을 방해하지 않는 범위에서만 사용한다.

## 말랑이 원칙
- 말랑이는 정답을 알려주는 캐릭터가 아니다.
- 학생이 스스로 찾을 수 있도록 힌트를 주는 학습 친구다.
- 속도나 정답만 칭찬하지 않는다.
- 도전, 집중, 끝까지 하기, 다시 시도하기, 표현하려는 시도를 칭찬한다.

## 활동 설계 원칙
- 같은 활동을 반복 생산하지 않는다.
- 책마다 핵심 개념을 먼저 분석하고, 그 책에 가장 적합한 활동 방식을 선택한다.
- 같은 조작 방식이라도 책마다 내용, 그림, 질문, 표현 활동은 달라져야 한다.
- 학생이 "공부한다"보다 "놀러 간다"고 느끼되, 놀다 보면 배움이 일어나야 한다.
- 기존 `빠질 때가 됐어` 활동 HTML은 1차 MVP 연결용이며, 다음 단계에서 위 원칙에 맞게 화면 문구와 활동 흐름을 재정리한다.

## 보안 체크리스트
- [x] 학생 화면에 관리자 진입 버튼을 만들지 않는다.
- [x] `/admin`은 직접 주소로 들어갈 때만 접근한다.
- [x] 1차 MVP 비밀번호는 실제 보안 기능이 아니라 데모용임을 코드와 화면에 표시한다.
- [x] Firebase API 키, 서비스 계정, 관리자 토큰 등 비밀값을 저장하지 않는다.
- [x] DB 쓰기/삭제 기능은 실제 연결 전까지 TODO로 둔다.
- [x] 사용자 입력값은 화면 확인용으로만 다루고 원격 저장하지 않는다.
- [x] 향후 Firebase 적용 시 Firestore Security Rules와 Firebase Authentication을 먼저 설계한다.
- [x] 공개/비공개 설정은 MVP에서 UI 상태만 제공하고 실제 접근 제어로 오해하지 않게 한다.
